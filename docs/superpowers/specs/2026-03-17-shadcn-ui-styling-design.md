# Shadcn/UI Styling Upgrade — Design Spec

**Date:** 2026-03-17
**Scope:** Style-only overhaul using shadcn/ui + next-themes. No functional changes.

---

## Goal

Replace the current custom CSS class-based styling with shadcn/ui components and Tailwind v4 utilities. Add dark mode support via `next-themes`. Extract Markdown typography styles to a dedicated file.

---

## Constraints

- Functionality must not change (routing, data loading, Three.js demos, Markdown rendering all remain identical)
- Tailwind CSS v4 (already installed) — use `pnpm dlx shadcn@canary` for v4 compatibility
- Package manager: pnpm

---

## New Dependencies

| Package | Purpose |
|---------|---------|
| `next-themes` | Dark / light mode ThemeProvider |
| shadcn/ui CLI (canary) | Component scaffolding (v4-compatible) |

**shadcn components to install:**
- `card` — home page demo cards
- `sidebar` — demo page navigation
- `button` — theme toggle
- `separator` — visual dividers

---

## Installation Steps

1. Run `pnpm dlx shadcn@canary init` — select style `new-york`, confirm Tailwind v4 mode
   - This creates `components.json` and `src/lib/utils.ts` (with `cn()` helper)
   - Verify the generated `globals.css` additions use bare `:root {}` blocks, not `@layer base`
2. Install components: `pnpm dlx shadcn@canary add card sidebar button separator`
3. Install next-themes: `pnpm add next-themes`

## Layout Architecture

### Root Layout (`layout.tsx`)

```
<html>
  <body class="pt-14">          ← padding compensates for fixed header
    <Providers>                  ← 'use client', wraps ThemeProvider
      <Header />                 ← fixed top, h-14, full-width
      {children}
    </Providers>
  </body>
</html>
```

> `ThemeProvider` lives inside `<Providers>` (`'use client'`), not directly in `layout.tsx`, to keep the root layout as a Server Component.

### Home Page (`page.tsx`)

```
<main> (max-width container, padded)
  ├─ Hero section (title + description, plain Tailwind utilities)
  └─ Demo grid (auto-fit columns)
       └─ <Link> wrapping <Card> × N
            └─ <CardHeader>
                 ├─ <CardTitle>
                 └─ <CardDescription>
```

### Demo Page (`/demos/[slug]/page.tsx`)

```
<SidebarProvider>
  <DemoSidebar currentSlug={slug} />      ← shadcn Sidebar
  <SidebarInset>
    <main>
      ├─ <header>  (demo title + description, Tailwind utilities)
      ├─ <section> (DemoCanvasShell — logic unchanged, className updated)
      └─ <section> (DemoMarkdown — unchanged)
    </main>
  </SidebarInset>
</SidebarProvider>
```

### DemoSidebar

Uses shadcn Sidebar primitives:
- `Sidebar` root with `collapsible="offcanvas"` (collapses to a slide-in drawer on mobile, matching the current stacked-layout behavior at breakpoint)
- `SidebarContent`
- `SidebarGroup` + `SidebarGroupLabel` ("Demos")
- `SidebarMenu` + `SidebarMenuItem`
- `SidebarMenuButton` with `isActive` prop for current demo highlight
- Props interface unchanged: `{ currentSlug: string }`

## globals.css After Changes

**Ordering requirement:** Run `pnpm dlx shadcn@canary init` first. The init command appends the shadcn `:root` / `.dark` CSS variable block (which provides `--background`, `--foreground`, `--font-sans`, etc.) to `globals.css`. Only after init has run should the manual removal of old rules take place, to avoid a broken intermediate state.

After init and cleanup, `globals.css` retains only:

```css
* { box-sizing: border-box; }
html, body { margin: 0; padding: 0; }
a { color: inherit; text-decoration: none; }
/* shadcn-generated :root / .dark CSS variable block appended below by init */
```

The current `body { color: #111827; background: #ffffff; font-family: Arial, Helvetica, sans-serif; }` is removed after `shadcn init` runs, as shadcn's CSS variable block (`--background`, `--foreground`, `--font-sans`) replaces these values.

Geist font CSS variables (`--font-geist-sans`, `--font-geist-mono`) continue to be injected by `layout.tsx`. Shadcn `--font-sans` is mapped to `var(--font-geist-sans)` in the shadcn-generated CSS block.

All `.home-*`, `.demo-*` custom classes are removed.
